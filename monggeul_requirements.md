## 회원관리

### REQ-AUTH-001 · 회원가입 `P0`

**기본 흐름**
- 사용자는 이메일·비밀번호·닉네임을 입력하여 회원가입할 수 있다.
- 회원가입 전 개인정보 수집·이용 동의 및 이용약관 동의가 필요하며, 미동의 시 가입이 불가하다.
- 필수 입력값과 필수 약관 동의가 모두 충족되면 회원가입을 완료할 수 있다.
- 회원가입 완료 시 토큰 5개가 지급된다.

**이메일**
- 이메일 형식(RFC 5321) 검증을 수행한다.
- 이메일 중복확인을 진행하며 이미 가입된 이메일로는 인증 코드를 발송할 수 없다.
- 중복확인 완료된 이메일로 무작위 영문 대문자 6자리 인증 코드가 발송되며, 3분 내로 입력해야 한다.
- 인증코드 발송 이후 30초 후 재발송 가능하며, 재발급 시 기존 코드는 즉시 만료 처리된다.

**비밀번호**
- 영어 대소문자·숫자·특수문자 중 최소 2종류 이상의 조합으로 이루어진 8~16자를 입력받으며 공백은 포함하지 않는다.
- 비밀번호 확인을 통해 입력된 비밀번호와 일치 여부를 확인한다.
- 비밀번호 확인 불일치 시 가입 불가하다.

**닉네임**
- 사용자는 서비스 내에서 사용할 닉네임을 설정할 수 있다.
- 한글·영문·숫자로 2~8자를 입력받으며 공백은 포함하지 않는다.

**선택 정보**
- 직업과 생년월일은 입력하지 않아도 회원가입이 가능하다.
- 직업은 목록으로 표시되며 수집한 직업 정보는 사용자 통계 분석 및 AI 학습에 활용된다.
- 생년월일은 년/월/일이 목록으로 표시되며 연령대별 분석과 생일 이벤트에 이용된다.

### REQ-AUTH-002 · 로그인 `P0`

**기본 흐름**
- 가입된 이메일·비밀번호로 로그인할 수 있어야 한다.
- 입력값이 공란일 경우 오류 메시지를 반환한다.
- 미가입 이메일 또는 비밀번호 불일치 시 오류 메시지를 반환한다.
- 로그인 실패 시 이메일 유지, 비밀번호 입력란이 초기화된다.
- 최초 로그인 시 캐릭터 생성 모달이 나타나며, 캐릭터를 생성하지 않으면 서비스 이용이 불가능하다.

**자동 로그인**
- 사용자의 기기 정보를 기반으로 서버에서 검증 가능한 암호화된 토큰(예: JWT, 세션)을 생성한다.
- 접속 시 저장된 토큰을 확인하고, 유효성 검증 성공 시 로그인 과정을 생략하고 로그인 처리된다.
- 사용자가 비밀번호를 변경하거나 로그아웃한 경우, 기존 기기의 자동 로그인 토큰을 무효화한다.
- 2주간 접속하지 않으면 자동 로그인 토큰을 무효화 처리한다.

### REQ-AUTH-003 · 소셜 로그인 `P1`

**기본 흐름**
- 사용자는 소셜 계정을 통해 간편 로그인/회원가입을 할 수 있다.
- 소셜 로그인 시 이메일을 수집하여 계정을 생성한다.
- 최초 소셜 로그인 시 추가 정보(닉네임, 직업, 생년월일) 입력이 가능하다.
- 이미 가입된 이메일과 동일한 소셜 계정으로 로그인 시 기존 계정과 연동 처리한다.
- 최초 로그인 시 캐릭터 생성 모달이 나타난다.

**지원 플랫폼**
- Kakao

**제약 사항**
- 소셜 로그인 사용자는 서비스 내 비밀번호가 별도로 설정되지 않으며, 비밀번호 재설정을 할 수 없다.

### REQ-AUTH-004 · 로그아웃 `P1`

- 사용자는 현재 로그인된 계정에서 로그아웃할 수 있다.
- 로그아웃 시 현재 로그인 세션이 종료되고, 랜딩페이지로 이동한다.

---

### REQ-AUTH-005 · 회원정보 수정 `P2`

**회원정보 수정**
- 마이페이지 진입 후 직업, 닉네임 수정이 가능하다.
- 닉네임 제약 조건은 REQ-AUTH-001과 동일.
- 현재 비밀번호 입력 후 일치 시 새 비밀번호로 수정이 가능하다.
- 비밀번호 수정 진행 시 비밀번호 제약 조건은 REQ-AUTH-001과 동일하다.
- 새 비밀번호 확인을 통해 입력된 새 비밀번호와 일치 여부를 확인한다.
- 비밀번호 수정 후 보안을 위해 로그아웃 처리된다.

**제약 사항**
- 소셜 로그인 회원의 경우 비밀번호 수정은 불가능하다.

---

### REQ-AUTH-006 · 회원 탈퇴 `P2`

- 사용자는 현재 비밀번호 입력 후 일치 시 회원 탈퇴를 진행할 수 있다.
- 소셜 로그인 사용자의 경우 현재 비밀번호 입력 없이 회원 탈퇴를 진행할 수 있다.
- 탈퇴 진행 시 재확인을 위해 "회원 탈퇴"를 입력해야 탈퇴 진행 가능하다.
- 탈퇴 완료 시 저장된 개인정보(원본이미지, 닉네임, 이메일, 비밀번호)는 삭제된다.
- 개인정보 이외의 이용했던 로그 정보는 삭제되지 않는다.

---

### REQ-AUTH-007 · 비밀번호 찾기 `P2`

- 비밀번호 분실 시 가입 이메일을 입력하여 인증 코드를 발급받을 수 있다.
- 미가입 이메일 입력 시 오류 메시지를 반환한다.
- 인증 코드는 무작위 영문 대문자 6자리로 이루어져 있으며 입력 가능 시간은 3분이다.
- 코드 재발급은 발송 이후 30초 후 가능하며, 재발급 시 기존 코드는 즉시 만료 처리된다.
- 새 비밀번호 제약 조건은 REQ-AUTH-001과 동일하다.
- 재설정 완료 시 사용된 코드 즉시 만료 처리 후 로그인을 진행할 수 있다.

## 캐릭터

### REQ-CHAR-001 · 이미지 업로드 `P1`

- 사용자가 원하는 이미지를 업로드할 수 있어야 한다.
- 이미지 업로드는 선택사항이며 이미지 업로드 없이 캐릭터 생성을 진행할 수 있다.
- 안내 문구: *"이미지를 업로드 하지 않는다면 입력된 성격 키워드와 캐릭터 설명에 따라 자동으로 생성됩니다."*

**제약 사항**
- 이미지 파일은 PNG, JPG, JPEG로만 업로드 가능하다.
- 이미지 파일의 크기는 5MB를 초과할 수 없다.
- 이미지 파일은 한 개만 업로드 가능하다.

---

### REQ-CHAR-002 · 페르소나 입력 `P0`

- 사용자는 성격 키워드 선택, 캐릭터 설명을 직접 입력하여 페르소나를 설정할 수 있다.
- 캐릭터 성격 키워드는 기본적인 성격들로 구성되어 있으며 이 중 최대 3개까지 선택할 수 있다.
- 캐릭터에 대한 설명은 자유 텍스트로 추가 성격을 묘사 가능하다.
- 설정된 페르소나는 퀘스트 생성, 알림, 피드 텍스트 생성에 반영된다.

**제약 사항**
- 캐릭터 설명은 한국어로 공백 포함 최대 200자까지 입력이 가능하다.

**성격 키워드 목록**
> 모험적인, 차분한, 호기심많은, 다정한, 장난스러운, 부지런한, 강력한, 몽환적인, 분노가 많은, 용감한, 온화한, 명량한

### REQ-CHAR-003 · 이미지 생성 `P0`

- 업로드한 이미지에 맞는 8비트 픽셀 이미지가 생성된다.
- 이미지 업로드를 하지 않은 경우 입력된 페르소나와 캐릭터 설명을 참고한 이미지가 생성된다.
- 사용자가 업로드한 이미지의 원본과 생성된 캐릭터 이미지는 저장된다.
- 생성된 캐릭터 이미지를 확인 후 확정 또는 재생성할 수 있다.
- 재생성 시도 시 이전에 입력했던 페르소나와 성격에 대한 내용은 유지되며, 수정 가능하다.

**제약 사항**
- 이미지 재생성 횟수는 계정당 1일 3회로 제한하고, 당일 최초 생성을 기준으로 24시간 이후 초기화된다.

---

### REQ-CHAR-004 · 캐릭터 등록 `P0`

- 사용자가 생성 결과를 확정하면 캐릭터가 마을 주민으로 등록된다.
- 캐릭터 등록 시 해당 캐릭터의 기본 집이 함께 생성된다.
- 캐릭터 등록이 완료되면 해당 캐릭터의 피드가 생성된다. (REQ-FEED-002 참조)

**제약 사항**
- 사용자는 최대 10명의 캐릭터를 보유할 수 있다.
- 캐릭터 보유 수가 최대치에 도달한 경우 추가 생성을 제한하고 안내 메시지를 표시한다.
  - 예: *"캐릭터 생성은 최대 10명까지만 가능합니다."*

---

### REQ-CHAR-005 · 캐릭터 목록 조회 `P1`

- 사용자는 생성된 캐릭터의 목록을 확인할 수 있다.
- 캐릭터 목록에는 캐릭터 이미지, 이름이 표시된다.
- 사용자는 캐릭터를 선택하여 상세 정보를 확인할 수 있다.

---

### REQ-CHAR-006 · 캐릭터 상세 조회 `P1`

- 사용자는 캐릭터 상세 조회를 통해 생성된 캐릭터의 이미지, 이름, 페르소나, 진행 중인 퀘스트 정보를 확인할 수 있다.

---

### REQ-CHAR-007 · 캐릭터 삭제 `P2`

- 사용자는 캐릭터를 삭제할 수 있다.
- 캐릭터 삭제 전 사용자에게 재확인을 요구한다.
  - 예: *"00캐릭터를 정말 삭제하시겠습니까?"*
- 삭제 시 해당 캐릭터의 피드, 게시물은 유지된다.
- 남아있던 퀘스트는 다른 캐릭터에게 재할당된다.

**제약 사항**
- 캐릭터는 최소 1명 이상 유지되어야 한다.

---

### REQ-CHAR-008 · 퀘스트 생성 `P0`

- 퀘스트는 사용자가 TODO 항목 확정 시 항목 하나당 랜덤한 캐릭터에게 생성된다.
- 생성된 퀘스트의 내용은 사용자의 TODO 내용과는 관련 없는, 캐릭터의 페르소나·외형과 관련된 독립 퀘스트이다.

**제약 사항**
- 퀘스트 생성 수는 하루 5개로 제한한다.

---

### REQ-CHAR-009 · 퀘스트 조회 `P1`

- 사용자는 캐릭터에게 부여된 퀘스트를 조회할 수 있다. (REQ-CHAR-006 참조)

---

### REQ-CHAR-010 · 퀘스트 완료 `P0`

- 사용자가 퀘스트와 연동된 TODO 항목을 완료하였을 때 퀘스트가 완료된다.
- 퀘스트 완료 시 토스트 알림과 함께 퀘스트 완료 여부를 확인할 수 있다.
  - 예: *"OO이가 피드를 업로드 했습니다"*

---

### REQ-CHAR-011 · 퀘스트 실패 `P1`

- TODO 항목을 삭제·포기하거나 미완료 상태로 초기화 시간(자정)이 되면 실패 처리된다.
- 사용자는 퀘스트 실패 처리 여부를 알림을 통해 확인할 수 있다.

## 일정 관리

### REQ-PLAN-001 · TODO 자동 생성 `P0`

**기본 흐름**
- 사용자는 프롬프트 입력란에 자연어로 여러 개의 할 일을 한 번에 입력할 수 있다.
- 당일의 일정 내용이 프롬프트에 자동 입력된다.
- 입력된 자연어를 LLM을 통해 task 단위로 나누어 자동으로 TODO 리스트를 생성한다.
- 사용자가 생성된 TODO 리스트를 확인하고 수정할 수 있다.
- 사용자가 최종적으로 리스트 내용을 확정하면 TODO 리스트에 반영된다.

> 프롬프트 입력 예시: *"오늘 운동하고 깃 커밋 하나 하고 기획서 작성할 거야."*
> TODO 생성 예시: 운동하기 / 깃 커밋하기 / 기획서 작성하기

**장기 플랜 생성**
- 사용자는 채팅 형식으로 대화하며 본인에게 맞는 플랜을 생성할 수 있다.
- 플랜 생성 중 필요한 정보가 부족한 경우 추가 질문이 생성된다.
- 추가 질문에 대한 답변을 참고하여 맥락과 함께 재분석한다.
- 플랜을 생성하여 사용자에게 제시한다.
- 생성된 플랜의 내용을 확인하고 수정 및 삭제가 가능해야 한다.
- 사용자가 플랜을 확정하면 캘린더에 해당 날짜에 맞는 일정이 자동 생성된다.
- 생성된 일정과 TODO에는 사용자 목표에 맞는 태그를 자동 부여한다.

**제약 사항**
- 플랜 생성 시 답변은 한국어로 공백 포함 최대 1500자 이내로 생성된다.
- 사용자 입력은 한국어로 입력받으며, 공백 포함 최대 600자까지 가능하다.
- TODO 생성 프롬프트 입력 시 공백 포함 200자 이내로 작성 가능하다.

> LLM 모델: GPT-4o 또는 파인튜닝 모델 사용 예정

### REQ-PLAN-002 · TODO 추가 `P0`

**제약 사항**

### REQ-PLAN-003 · TODO 완료 `P0`

- 사용자는 TODO 리스트에서 완료한 항목에 대한 완료 체크를 진행할 수 있다.
- TODO 완료 체크 시 해당 퀘스트가 부여된 캐릭터의 퀘스트도 완료된다.

**토큰 지급**
- TODO 항목 하나 완료 시 토큰이 1개씩 지급된다.
- 하루 토큰 지급 상한선: 10개

### REQ-PLAN-004 · TODO 실패 `P2`

- TODO 리스트와 캐릭터의 퀘스트는 매일 자정에 실패 처리된다.
- 실패 처리된 TODO 항목은 토큰을 소모(4개)하여 다음날 자정까지 연장할 수 있다.
- 사용자는 완료 처리가 안된 항목에 대해 스스로 실패 처리가 가능하다.

**제약 사항**
- TODO 기간 연장은 항목당 1회만 가능하다.

### REQ-PLAN-008 · 일정 추가 `P0`

- 사용자가 캘린더에서 특정 날짜를 선택하여 직접 일정을 입력할 수 있다.
- 일정 생성 시 제목, 날짜, 설명, 태그를 입력할 수 있다.
- 일정과 TODO는 라디오박스를 통해 구분할 수 있다.
- 생성 완료 시 해당 날짜의 캘린더에 해당 내용이 반영된다.

**제약 사항**
- 제목 입력은 한국어로 공백 포함 최대 20자까지 가능하다.
- 설명 입력은 한국어로 공백 포함 최대 200자까지 가능하다.

> 일정: 캘린더에 일자로 구분되어 추가된 이벤트 (기간)
> TODO: 하루에 해야하는 일 (리스트)

### REQ-PLAN-009 · 일정 조회 `P0`

- 캘린더에서 날짜별 TODO 리스트와 일정을 확인할 수 있다.

### REQ-PLAN-010 · 일정 상세 조회 `P0`

- 상세 내용은 제목, 날짜, 설명, 태그로 이루어져 있다.

### REQ-PLAN-011 · 일정 수정 `P2`

- 사용자는 등록된 일정의 제목, 날짜, 설명, 태그를 수정할 수 있다.

### REQ-PLAN-012 · 일정 삭제 `P2`

## 회고

### REQ-LOGS-001 · 하루 회고 작성 `P1`

- TODO 모두 완료 시 또는 미완료 상태에서는 매일 자정에 하루 회고 작성 토스트 알림이 표시된다.
- 회고 작성은 다음날 자정까지 가능하며, 그 이후에는 해당일에 대한 회고 작성이 불가능하다.
- 회고는 잘한 점, 개선점을 구분하여 입력 가능하다.
- 회고는 작성 후 확정 여부를 재확인하며 확정할 수 있다.
- 작성된 회고는 각 날짜별로 구분되어 저장된다.
- 회고 작성 완료 시 작성 항목당 토큰 2개가 지급된다.

**제약 사항**
- 회고 입력은 최대 400자로 제한한다.
- 회고 작성 시 공백 포함 30자 이상 작성하지 않는다면 회고 작성은 가능하지만 토큰 획득은 불가능하다.
- 회고 작성 후 제공되는 토큰은 일일 수량 제한에 해당하지 않는다.

## SNS

### REQ-FEED-001 · 타임라인 조회 `P0`

- 게시물의 내용과 게시물을 올린 캐릭터의 프로필 사진과 이름을 확인할 수 있다.

### REQ-FEED-003 · 피드 조회 `P0`

- 각 캐릭터의 피드를 조회할 수 있다.
- 캐릭터의 피드에서 해당 캐릭터의 게시물 목록을 조회할 수 있다.

### REQ-FEED-004 · 게시물 생성 `P0`

- TODO 완료 시 피드에 캐릭터의 게시물이 생성된다.
- 게시글은 캐릭터의 퀘스트에 해당하는 내용과 페르소나에 관련된 말투로 140자 이내의 한국어로 생성된다.
- 게시물 생성 시 이미지 포함 여부는 랜덤으로 결정된다.
- 이미지는 캐릭터의 퀘스트와 관련된 내용으로 생성된다.

**제약 사항**
- 사용자당 이미지가 포함된 게시물 생성 가능 수는 하루 최대 5개이다.

---

### REQ-FEED-005 · 게시물 조회 `P1`

- 각 게시물마다 프로필 사진, 이름, 게시물 사진(존재 시), 게시글, 좋아요 수, 댓글을 확인할 수 있다.
- 각 게시물에 좋아요를 표시할 수 있으며, 다시 한번 누르면 취소된다.

---

### REQ-FEED-006 · 게시물 삭제 `P2`

- 삭제 시도 시 확인 팝업이 나타나며, 삭제 후에는 되돌릴 수 없다.

---

### REQ-FEED-007 · 댓글 작성 `P1`

- 사용자는 해당 게시물에 토큰을 소모(3개)하여 댓글을 작성할 수 있다.
- 작성한 댓글은 수정 불가능하다.
- 댓글 작성은 하루 최대 5개로 제한한다.

### REQ-FEED-009 · 답글 생성 `P1`
