## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 몽글마을 (내일도와줘) |

| 핵심 가치 | 할일 관리를 게임처럼 즐겁게, 캐릭터와의 애착을 통해 지속적인 동기부여 |

> 사용자가 직접 만든 AI 캐릭터가 마을에 살면서 퀘스트를 수행하고, 사용자의 할일 완료가 캐릭터의 성장과 연결되는 To-do 게임화 서비스

## 2. 핵심 콘셉트

### 2-1. AI 애착 캐릭터

- 사용자가 이미지를 업로드하거나 키워드를 입력하면 **8bit 픽셀 스타일**의 고유 캐릭터가 생성된다.
- 캐릭터는 성격 키워드와 자유 텍스트 설명으로 이루어진 **페르소나**를 가진다.
- 페르소나는 퀘스트 내용, 피드 게시글 말투, 응원 멘트 등 서비스 전반에 반영된다.
- 최대 10명의 캐릭터를 보유할 수 있으며, 각각 마을에 집과 피드를 가진다.

### 2-2. 퀘스트와 TODO의 분리

- 사용자의 TODO와 캐릭터의 퀘스트는 **내용상 독립적**으로 설계된다.
- 사용자가 TODO를 완료하면 → 연동된 캐릭터의 퀘스트도 완료되는 구조.
- 퀘스트 내용은 캐릭터의 페르소나·외형 기반으로 생성되어 몰입감과 애착을 강화한다.
- 예: 사용자의 TODO가 "기획서 작성"이라도, 캐릭터의 퀘스트는 "숲속 탐험을 떠나기"일 수 있다.

### 2-3. 마을 맵 기반 UI

- 서비스의 메인 화면은 스타듀밸리 스타일의 **마을 맵**이다.
- 캐릭터들이 마을을 배회하며 말풍선으로 퀘스트 상태를 표시한다.
- 집 외형, 마당 등을 토큰을 소모해 커스터마이징할 수 있다.

## 3. 서비스 전체 플로우

### 플로우 개요 (10단계)

```
로그인 → 마을 맵(메인) → 캐릭터 생성 → TODO 생성 → 계획 완료
→ 일정 관리 챗봇 → 캘린더 → 회고 유도 → 커스터마이징 → 설정
```

---

### STEP 1. 회원가입 / 로그인

- 이메일·비밀번호 기반 회원가입 또는 **카카오 소셜 로그인** 지원
- 이메일 인증코드(영문 대문자 6자리, 유효 3분) 방식
- 최초 로그인 시 **캐릭터 생성 모달** 강제 진입 → 캐릭터 없으면 서비스 이용 불가
- 자동 로그인 지원 (JWT 기반, 2주 미접속 시 만료)
- 회원가입 완료 시 토큰 5개 지급

---

### STEP 2. 메인화면 (마을 맵)

- 스타듀밸리 스타일의 마을 맵이 메인 화면
- 주요 구성 요소:
  - **이장님 집**: 채팅 선택지 진입점 (일정 관리 챗봇)
  - **캐릭터 배회**: 캐릭터들이 마을을 돌아다니며 말풍선으로 퀘스트 표시
  - **포모도로 타이머**: 햇님 아이콘 형태, 25분 집중 / 5분 휴식
  - **오늘의 TODO 리스트**: 메인에서 바로 확인 및 체크 가능
  - **토스트 알림**: 퀘스트 완료, 피드 업로드, 회고 유도 등

---

### STEP 3. 캐릭터 생성

- **이미지 업로드** (PNG/JPG/JPEG, 최대 5MB, 1개) 또는 미업로드 선택
- **성격 키워드** 최대 3개 선택
  - 선택지: 모험적인, 차분한, 호기심많은, 다정한, 장난스러운, 부지런한, 강력한, 몽환적인, 분노가 많은, 용감한, 온화한, 명량한
- **캐릭터 설명** 자유 텍스트 입력 (최대 200자)
- LLM이 입력값을 기반으로 **8bit 픽셀 이미지** 자동 생성
- 생성 결과 확인 후 확정 또는 재생성 (재생성 일 3회 제한)
- 확정 시 캐릭터가 마을 주민으로 등록, 기본 집 생성, 피드 생성

---

### STEP 4. TODO 생성

- 사용자가 자연어로 할 일을 프롬프트 입력 (최대 200자)
  - 예: *"오늘 운동하고 깃 커밋 하나 하고 기획서 작성할 거야."*
- LLM이 task 단위로 분해하여 TODO 리스트 자동 생성
  - 예: 운동하기 / 깃 커밋하기 / 기획서 작성하기
- 당일 캘린더 일정이 프롬프트에 자동 포함
- 생성된 리스트 수정 후 확정 → TODO 리스트에 반영
- 직접 추가도 가능 (최대 20자)
- TODO 확정 시 랜덤 캐릭터에게 **독립 퀘스트 자동 생성** (일 5개 제한)

---

### STEP 5. 계획 완료 (TODO 체크)

- TODO 항목 체크 시:
  - 연동된 캐릭터 퀘스트 완료 처리
  - 캐릭터 피드에 게시물 자동 생성 (퀘스트 기반 내용, 페르소나 말투, 140자 이내)
  - 이미지 포함 여부 랜덤 결정 (하루 최대 5개)
  - 토큰 1개 지급 (일일 상한 10개)
  - 토스트 알림 표시
- 자정에 미완료 항목은 실패 처리 (토큰 4개 소모 시 1회 연장 가능)

---

### STEP 6. 일정 관리 챗봇

- 이장님 집에서 진입하는 **멀티턴 LLM 챗봇**
- 사용자와 대화하며 장기 플랜 생성
- 정보 부족 시 추가 질문 생성 → 답변 반영하여 재분석
- 플랜 확정 시 캘린더에 해당 날짜 일정 자동 생성 + 태그 자동 부여
- 입력 제한: 600자 / 답변 제한: 1500자

---

### STEP 7. 캘린더

- TickTick 기준 UI 레퍼런스
- 날짜별 TODO 리스트와 일정(이벤트) 구분 표시
- 프로젝트 태그 색상으로 구분
- 연속 일정 표시 지원
- 직접 일정 추가/수정/삭제 가능 (제목 20자, 설명 200자)

---

### STEP 8. 회고 유도

- TODO 전체 완료 시 또는 자정에 회고 작성 토스트 알림 표시
- 회고 작성 가능 시간: 다음날 자정까지
- 잘한 점 / 개선점 구분 입력 (각 최대 400자)
- 30자 미만 작성 시 저장은 가능하나 토큰 미지급
- 완료 시 항목당 토큰 2개 지급 (일일 상한 미적용)
- 수정은 토큰 15개 소모

---

### STEP 9. 커스터마이징

- 획득한 토큰을 소모하여 캐릭터의 집 외형, 마당 변경
- 커스터마이징 비용은 **프롬프트 길이에 비례**하여 소모 (자원 남용 방지)

---

### STEP 10. 설정

- 닉네임·직업 수정, 비밀번호 변경, 회원 탈퇴
- 캐릭터 관리 (목록 조회, 상세 조회, 삭제)
- 토큰 잔액 및 획득/사용 이력 조회
- 알림 기록 조회 및 삭제
- 채팅 로그 확인

## 4. UI/UX 콘셉트

### 비주얼 방향성

| 항목 | 내용 |
|------|------|
| 전체 스타일 | 스타듀밸리 스타일 8bit 픽셀 아트 |
| 캐릭터 이미지 | 8bit 픽셀, 이미지 미업로드 시 LLM 자동 생성 |
| 메인 화면 | 마을 맵 (탑뷰 형태) |
| 타이머 아이콘 | 햇님 아이콘 (포모도로) |
| 알림 방식 | 토스트 알림 |

### 주요 UI 레퍼런스

- **스타듀밸리**: 마을 맵, 캐릭터 배회, 집 커스터마이징, 8bit 비주얼 전반
- **TickTick**: 캘린더 UI 구성 (날짜별 TODO, 태그 색상, 연속 일정)

### 핵심 인터랙션

- 캐릭터가 마을을 **배회**하며 말풍선으로 퀘스트 상태 표시
- TODO 완료 시 **즉각적인 피드백** (토스트 알림 + 피드 게시물 생성)
- 회고 모달은 완료 또는 다음날 접속 시 **자연스럽게 유도**

## 5. LLM 활용 구조

> 모델: GPT-4o 또는 파인튜닝 모델 예정

| 기능 | LLM 역할 | 입력 | 출력 |
|------|----------|------|------|
| 캐릭터 이미지 생성 | 8bit 픽셀 이미지 생성 | 업로드 이미지 or 페르소나 키워드+설명 | 8bit 픽셀 캐릭터 이미지 |
| TODO 자동 생성 | 자연어 → task 분해 | 사용자 자연어 입력 (200자) + 당일 일정 | TODO 리스트 |
| 퀘스트 생성 | 캐릭터 독립 퀘스트 생성 | 캐릭터 페르소나 + 외형 정보 | 퀘스트 텍스트 |
| 피드 게시물 생성 | 캐릭터 말투 게시글 생성 | 퀘스트 내용 + 페르소나 | 140자 이내 한국어 게시글 |
| 피드 이미지 생성 | 퀘스트 관련 이미지 생성 | 퀘스트 내용 | 게시물 첨부 이미지 |
| 일정 관리 챗봇 | 멀티턴 플랜 생성 | 사용자 대화 (600자) + 히스토리 | 장기 플랜 + 캘린더 일정 |
| 답글 생성 | 캐릭터 대댓글 생성 | 댓글 내용 + 페르소나 | 캐릭터 답글 (10분 후) |
| 포모도로 응원 멘트 | 캐릭터 응원 문구 생성 | 페르소나 | 휴식 시간 토스트 멘트 |

## 6. 토큰 경제 설계

토큰(사과)은 계정 단위로 통합 관리되며, 완료 행동에 보상을 연결하되 일일 상한선으로 남용을 방지한다.

### 획득

| 행동 | 지급량 | 일일 상한 | 비고 |
|------|--------|-----------|------|
| 회원가입 완료 | +5개 | - | 최초 1회 |
| TODO 항목 완료 | +1개/항목 | 10개 | - |
| 회고 작성 (잘한 점) | +2개 | 상한 미적용 | 30자 이상 작성 시 |
| 회고 작성 (개선점) | +2개 | 상한 미적용 | 30자 이상 작성 시 |

### 소모

| 행동 | 소모량 | 제한 |
|------|--------|------|
| TODO 기간 연장 | 4개/항목 | 항목당 1회 |
| 댓글 작성 | 3개 | 하루 최대 5개 |
| 회고 수정 | 15개 | - |
| 커스터마이징 | 프롬프트 길이 비례 | - |

### 설계 원칙

- **완료 행동에 보상 연결**: 할일 완료와 회고라는 핵심 행동에 집중하여 동기부여
- **일일 상한선**: TODO 토큰은 일 10개 상한으로 반복 남용 방지
- **회고는 상한 미적용**: 회고의 질적 가치를 높이기 위해 별도 처리
- **커스터마이징 비용 비례**: 프롬프트 길이에 따라 소모량이 달라져 자원 소모 균형 유지

## 7. 캐릭터 시스템

### 캐릭터 생성 흐름

```
이미지 업로드(선택) + 성격 키워드(최대 3개) + 캐릭터 설명(200자)
→ LLM이 8bit 픽셀 이미지 생성
→ 확정 or 재생성 (일 3회 제한)
→ 마을 주민 등록 + 집 생성 + 피드 생성
```

### 페르소나의 역할

페르소나는 서비스 전반에 걸쳐 캐릭터의 정체성을 유지하는 핵심 요소다.

| 활용 위치 | 내용 |
|-----------|------|
| 퀘스트 생성 | 캐릭터 성격에 맞는 독립 퀘스트 내용 결정 |
| 피드 게시글 | 캐릭터 말투와 관점으로 게시글 작성 |
| 포모도로 응원 | 캐릭터 성격에 맞는 응원 멘트 |
| 답글 생성 | 댓글에 캐릭터 페르소나로 반응 |

### 퀘스트 설계 원칙

- 캐릭터 퀘스트는 사용자 할일과 **내용상 독립적**
- TODO 완료라는 행동만 연동 → 사용자가 할일을 완료해야 캐릭터도 퀘스트 달성
- 이 분리 구조가 캐릭터에 대한 **애착과 몰입감**을 강화

### 제약 사항 요약

| 항목 | 제한 |
|------|------|
| 최대 캐릭터 수 | 10명 |
| 퀘스트 생성 | 하루 5개 |
| 이미지 재생성 | 하루 3회 (24시간 기준 초기화) |
| 최소 캐릭터 수 | 1명 (삭제 시 최소 1명 유지) |

## 8. 성공 기준

서비스 플로우 전반이 유기적으로 작동하는 완성도 있는 웹앱 구현.

### 핵심 플로우 체크리스트

- [ ] 로그인 → 마을 맵 진입
- [ ] 최초 로그인 시 캐릭터 생성 강제 진입 및 완료
- [ ] TODO 자연어 입력 → LLM 분해 → 리스트 확정
- [ ] TODO 완료 → 캐릭터 퀘스트 완료 → 피드 게시물 자동 생성
- [ ] 자정 실패 처리 및 토큰 연장 동작
- [ ] 일정 관리 챗봇 → 캘린더 일정 자동 생성
